### Check 1 — Router/Registry Mismatch (RED)

`assemble_crew()` at `crews.py:2751` uses `CREW_REGISTRY.get(crew_type, build_unknown_crew)`. The following task types route to crew keys that do not exist in CREW_REGISTRY — any matching request silently degrades to unknown_crew:

| Task type | Router crew value | CREW_REGISTRY status |
|---|---|---|
| `website_build` | `web_builder_crew` | Missing — registry has `website_crew` |
| `app_build` | `app_builder_crew` | Missing — registry has `app_crew` |
| `voice_polishing` | `voice_crew` | Missing — registry has `voice_polisher_crew` |
| `linkedin_x_campaign` | `griot_crew` | Missing — registry has `linkedin_x_crew` |
| `notion_task` | `notion_crew` | Completely absent from registry |
| `practice_runner_task` | `practice_runner_crew` | Completely absent from registry |
| `skill_build` | `skill_builder_crew` | Completely absent from registry |
| `chat` | `chat_crew` | Completely absent from registry |

**Fix:** In `router.py`, rename the four name-mismatch entries:
- `web_builder_crew` → `website_crew`
- `app_builder_crew` → `app_crew`
- `voice_crew` → `voice_polisher_crew`
- `griot_crew` → `linkedin_x_crew`

For the four completely missing crews (`notion_crew`, `practice_runner_crew`, `skill_builder_crew`, `chat_crew`): either add corresponding builder functions to `crews.py` and register them in CREW_REGISTRY, or remove the task types from `router.py` if those features are deprecated.
