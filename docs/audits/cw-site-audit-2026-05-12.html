Legacy .html redirect inconsistency (low-priority warning)

The .htaccess rule appends .html for paths without an extension, which makes /governance and /governance.html BOTH return 200. The sitemap lists the .html versions; the canonical tags on the lens/for pages reference clean URLs (/lens/throughput without .html). Internal inconsistency. Google will pick one as canonical based on signals it sees, but it adds noise.

Suggested fix: either (a) add a 301 redirect rule for (.+)\.html$ → /$1 to enforce clean URLs as the only public form, or (b) standardize sitemap to clean URLs and add explicit <link rel="canonical"> on the root-level pages too. Not blocking.

Critical: /capture endpoint is broken

P0 · revenue-impacting · broken since 2026-05-11 deploy

The "save this as a PDF / send me my diagnosis" form on the diagnostic demo sends every submission to a non-existent webhook.

Front-end POSTs to https://n8n.srv1040886.hstgr.cloud/webhook/catalystworks-constraints-ai/capture. n8n returns:

{
  "code": 404,
  "message": "The requested webhook \"POST catalystworks-constraints-ai/capture\" is not registered.",
  "hint": "The workflow must be active for a production URL to run successfully."
}

The front-end (index.html:2222) explicitly swallows this error and shows the user a success message regardless: "Still show success — we don't want to make the user feel like it failed".

Consequence: Every prospect who has handed over their email since 2026-05-11 has been told it was captured. None of them landed in Supabase. None of them are getting a follow-up. SELECT COUNT(*) FROM diagnostic_captures = 0.

The fix

Full design lives in agent_outputs/outreach/n8n_constraints_integration.md. Summary:

  • Recommendation: deploy the existing _worker.js Cloudflare Worker (code already written, undeployed) and point the front-end at it. Worker writes to Supabase diagnostic_captures AND POSTs to agentsHQ /inbound-lead, which triggers the new constraints_ai 3-email sequence.
  • n8n retirement: the Worker's LLM-call path is also already written. Once deployed, n8n becomes redundant for this pipeline — one less SaaS, one less failure surface, code-as-truth instead of UI-as-truth.
  • 3-email follow-up: templates shipped in this same branch — templates/email/constraints_ai_t1.py, t2.py, t3.py. Cadence Day 0/2/4 (warm inbound). Wired into skills/outreach/sequence_engine.py as pipeline='constraints_ai'.
  • Sequence stays draft-only until Boubacar explicitly flips AUTO_SEND_CONSTRAINTS_AI=true. No emails fire without explicit go-ahead per the hard CLAUDE.md rule.
