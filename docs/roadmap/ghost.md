## Why This Is Separate

`harvest.md` = the existing CW revenue motion (Signal Sessions + Signal Works infrastructure + first paid contract). `ghost.md` = a parallel validation track for a NEW SKU. Reasons it gets its own roadmap (per Sankofa Council 2026-05-11):

- Different ICP (founder/exec with voice already, no time → vs SW = local service business needs presence)
- Different deliverable (weekly ghostwritten posts on retainer → vs SW = site/funnel/lead-gen one-time + monthly)
- Different sales cycle (low-commit $2K/mo trial-as-sample → vs SW = $497 setup + ongoing infrastructure)
- Different fail mode (GW fails if voice clone produces slop; SW fails if site doesn't convert)

**Bidirectional ladder with SW** (Boubacar 2026-05-11):
- SW client → upsell GW retainer ("we built your funnel, want us writing the posts too?")
- GW client → upsell SW infrastructure ("posts landing — want the funnel behind them?")
- Bundle: SW + GW = $2.5K/mo "AI presence stack"

If validation succeeds, GW merges into `harvest.md` as Tier 3. Until then, ghost.md tracks the experiment separately so harvest stays focused on CW Signal Sessions + SW first contract.
