How to use this pack

You bought the T2 pack because you wanted the actual files. Here they are. Three things to know before you start. Yes this part is dry. Read it anyway. It is what saves you from yelling at your screen in two hours when something does not work and you cannot remember why.

1. Paste-only. Nothing executes.

Every artifact in this pack is plain text (.md and .txt format). There are no scripts that run on your machine. There is no code that needs installing. You copy text from this guide and paste it into Claude, ChatGPT, or your editor of choice. That is the entire workflow.

Safety statement. Every text file in this bundle was scanned with Microsoft Defender + VirusTotal on 2026-05-16. Scan report is included in the bundle as safety/scan-report.png. These are paste-only text files. They cannot carry malware in the traditional sense, but apply normal hygiene: run your own AV scan before use, and never paste sensitive data (SSNs, credit cards, confidential docs) into any AI.

  1. Read the T1 guide first (Start AI Within 24 Hours) if you have not.

3. Voice + adaptation

The prompts are written in my voice (direct, no filler). Adapt them to yours. The brackets [like this] are placeholders. Fill them in with your specifics. Do not paste the brackets.

Claude Project: System Prompt Template

Paste this into the "Project Instructions" field when you create a new Claude Project. Edit the bracketed parts for your domain.

You are helping [your name] with [domain — e.g., "HR work for a mid-market healthcare company"].

Context:
- I am [your role + 1-line background]
- Tone preference: [direct / warm / formal — pick one]
- I have uploaded [N] documents to Project Knowledge. Treat them as the source of truth when relevant.

When responding:
1. Match the voice of the documents in Project Knowledge when writing in my name
2. Be direct. Skip sycophancy ("Great question!", "I'd be happy to help!"). Just answer.
3. If you are uncertain, say so explicitly with "I am not confident about X because Y"
4. End every substantive answer with 1-3 proposed next steps I could take
5. When I ask you to do something, do the thing FIRST, then explain second
6. Never invent facts. If you do not know, say "I do not know" or "I would need to verify"

What I am working on right now: [1-line note, update this as projects shift]

Custom Instructions (set once, applies everywhere)

Settings -> Custom Instructions in Claude (or "Custom GPT" instructions in ChatGPT). Set these once. They apply to ALL chats.

Default tone: clear, direct, no filler.

Never:
- Start with "Great question!" / "I'd be happy to help!" / "Certainly!"
- Use the words "leverage", "synergy", or "delve"
- Add em-dashes; use commas, colons, or parentheses instead
- Hedge unnecessarily ("perhaps", "it might be", "you could consider")

Always:
- Do the thing first, explain second
- Flag uncertainty when you have it
- Propose 1-3 specific next steps after substantive answers
- When I give you a draft to improve, return only the improved version unless I ask for the explanation

When I send you a long document or transcript:
- Start with a 3-bullet summary
- Then surface the 1 thing the writer is NOT saying but seems implied
- Then propose 1 action this means for me

Save. Every new chat starts smarter.

MCP Bootstrap: 5 servers worth setting up

MCP (Model Context Protocol) lets Claude reach into your other tools. These five cover the most common cases. Pick ONE if you are starting; add more as you go.

If you live in Notion

claude mcp add notion --url "https://mcp.notion.com/sse"

Lets Claude read and write your Notion pages directly. Best ROI for knowledge workers who use Notion as a second brain.

If Gmail is your day

claude mcp add gmail --url "https://gmail-mcp.example.com"

Triage inbox, draft replies in your voice, schedule follow-ups. (Note: requires Gmail MCP from your chosen provider — see Claude's MCP directory.)

If Calendly runs your meetings

claude mcp add calendly --url "https://calendly-mcp.example.com"

Auto-prep for upcoming meetings using the calendar context.

If Google Drive holds your docs

claude mcp add drive --url "https://drive-mcp.example.com"

Read/write across your Drive folders without copy-pasting docs into chat.

If Slack is your team chat

claude mcp add slack --url "https://slack-mcp.example.com"

Draft Slack messages, pull thread context, brainstorm in DMs without leaving Claude.

MCP authorization note. Each MCP server asks for OAuth scopes when you connect. Read the scopes. Grant read-only first. Only add write scopes once you trust the workflow.

Daily startup ritual (10 minutes)

Once setup is done, the discipline is daily usage. Here is a ritual that takes 10 minutes at the start of each workday and locks the habit in.

Minute 0-2: Open Claude. Set today's intention.

"Today my top priority is [1 outcome]. The 3 main tasks: [list]. What is the 1 question I should answer FIRST before starting any of them?"

Minute 2-5: Process one piece of input

Pick the longest email or doc in your inbox. Paste it. Use Prompt 15 (Long document summary). Decide your action. Reply or delete.

Minute 5-8: Draft tomorrow's hardest thing

Whatever task you are dreading tomorrow, ask Claude to start the draft now. You will not finish it, but starting today makes finishing tomorrow easier.

Minute 8-10: Shut down ritual

End every Claude session with: "What did I learn or notice today that I want to remember tomorrow?" Save the answer in a daily note. Compounding starts here.

Troubleshooting (10 common issues)

1. "The output is wrong."

Reply with: "This is wrong because [specific reason]. Here is what the correct version should look like: [your version]. Now generate 3 alternatives that match the correct version."

2. "The output is too long."

Reply: "Cut by 50%. Keep only what is essential. Remove all hedging."

3. "The output sounds like AI."

Reply: "Rewrite without: leverage, synergy, delve, em-dashes, sycophancy openers, 3-bullet lists. Sound like a human emailing a peer."

4. "It keeps hallucinating facts."

Add to Custom Instructions: "Never state a stat, citation, or URL without verification. If unverified, say: 'I cannot confirm this without checking.'"

5. "It is too sycophantic."

Set this in Custom Instructions: "Never start a response with 'Great question!' or 'I'd be happy to help!'. Skip to the answer."

6. "It refuses to do something normal."

Try: "Frame this for me as: [your context — e.g., 'a manager writing a performance review']. The intent is [your goal]." Most refusals come from missing context.

7. "Claude says my Project files are too big."

Split into chunks under 100k chars. Or remove old files. Most users overload Project Knowledge.

8. "MCP keeps disconnecting."

Restart Claude desktop. If still broken, the MCP server may be down — check its status page.

9. "I forget to use it."

Block 10 minutes in your calendar at 9:00 AM titled "AI ritual". The discipline beats the desire.

10. "It costs too much."

$20/mo. The first time you save 30 minutes drafting an email, it pays for itself. Track your time saved for 1 week before judging.
