01
Why most agent stacks fail

Demos are easy. Stacks that survive Monday are not.

You have seen the demos. The one where an agent autonomously writes a function, commits it, and the founder smiles. You have probably also tried building one. By week 3 your agent destroyed its own context, lost a session of work, or merged a branch that broke main and you spent the next two days untangling it. That is not a tool problem. That is an architecture problem.

agentsHQ has 75 skills, multiple concurrent sessions, a gate for shared-state writes, a pre-commit hook stack, and three brands running on it. It works because every piece is named, scoped, and constrained. The playbook walks through the actual decisions: governance shape, audit registry, hard rules, why worktree-only edits, commit-per-fix, what to put behind the gate and what to let run loose.

This is not "5 frameworks for AI agents." This is one operator's stack. I describe what is in mine, why it is in mine, and what to steal for yours.

The boring rules are the load-bearing part. Worktree discipline does not look like a feature. It is the reason main has not been broken in six months. What I learned the hard way

05
Six honest questions

Before you buy.

Q1Do I get the agentsHQ code?
No. You get the playbook: architecture, rules, patterns, RCAs, and decisions. The code itself is private. Most of the value is in the shape, not the specific Python. You can build your own with the patterns inside.
Q2Do you push a specific LLM?
No. The architecture is model-agnostic. I run Claude as the primary because it fits my work. The playbook explains the trade-off honestly. ChatGPT, Gemini, and local models work with the same patterns.
Q3Is this for solo builders or teams?
Both. The system was built for one operator (me) running multiple sessions. It scales to small teams because the gate pattern works the same way. Larger orgs (10+ engineers) need the next layer up, which the playbook gestures at but does not solve.
Q4How long is the playbook?
Roughly 120 pages of dense PDF. Plus an HTML web version. Plus a markdown copy. Plus the raw source files referenced (CLAUDE.md, AGENT_SOP.md skeletons, pre-commit hook stubs). Not a 30-min skim.
Q5Do you keep updating it?
Yes. Major revision quarterly. Lifetime access for buyers. Patterns evolve as agentsHQ evolves. You get the new version when I ship.
Q6What if my stack is different?
The patterns translate. Worktree discipline applies to any git-backed agent system. Audit registry applies anywhere multiple agents touch shared state. The playbook describes shapes, not tools, on purpose.
