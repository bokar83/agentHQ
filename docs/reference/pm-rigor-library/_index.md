# Pattern Index :  Trigger to Pattern

Quick lookup for "what pattern fires when I see X."

## By trigger phrase

| If you hear / write / think... | Pull pattern |
|---|---|
| "What are we trying to accomplish?" | objective-statement |
| "How will we know we're done?" / "What does good look like?" | completion-criteria |
| "We can't do all of this by [date]" | constraint-matrix |
| "Scope is creeping" / "We added something else" | constraint-matrix |
| "Lots of ideas, can't process them all now" | parking-lot |
| "What could go wrong?" | risk-brainstorm |
| "Should we even keep going?" / "Time for a go/no-go" | stargate-checkpoint |
| "Where are we vs. plan?" / "What's the status?" | mitar-loop |
| "We're done. What did we learn?" | closeout-meeting |

## By engagement type

### Signal Session ($497, 90 min)

**Always pull:** objective-statement, completion-criteria, constraint-matrix (light), parking-lot
**Pull at end:** closeout-meeting (5-min version)
**Skip:** risk-brainstorm (parking-lot is enough), stargate-checkpoint (no future gate), mitar-loop (one session)

### SHIELD kickoff (multi-week, $20K+)

**Always pull at kickoff:** objective-statement, completion-criteria, constraint-matrix (full), risk-brainstorm, parking-lot
**Pull at midpoint:** stargate-checkpoint (decide whether to pivot or stay the course)
**Ongoing:** mitar-loop (light version, weekly)
**Pull at end:** closeout-meeting (full)

### Advisory retainer (recurring, monthly)

**Pull at start of relationship:** objective-statement, completion-criteria
**Ongoing:** mitar-loop (monthly check)
**Pull at quarterly review:** stargate-checkpoint, closeout-meeting (mini)

### Internal build (uses roadmap skill)

**Pull at kickoff:** objective-statement, completion-criteria, constraint-matrix (full)
**During:** parking-lot (in roadmap session log)
**At decision points:** stargate-checkpoint, constraint-matrix re-read
**At end:** closeout-meeting

### Content / writing project

**Pull at start:** objective-statement (the 25-word clarity test is brutal for content goals)
**Pull at end:** closeout-meeting (mini, 1 paragraph each section)

## By size of work

| Size | Default rigor pull |
|---|---|
| < 1 hour | objective-statement only (or nothing) |
| 1 day to 1 week | objective-statement + completion-criteria |
| 1 week to 1 month | + constraint-matrix + parking-lot |
| 1 month + | + risk-brainstorm + mitar-loop + stargate-checkpoint |
| Always at end | closeout-meeting (scaled to size) |

## Overrides

If the user says "pull X pattern" by name, do that, regardless of defaults. If the user says "skip the matrix this time," do that.

The defaults are scaffolding for when nobody specifies. The user's explicit pull always wins.
