# Eval Rubric: "Humanized Standard"
**agentsHQ: Design Quality Gate**
*Used by the critic agent after every build. Also used for baseline measurement and test_log.md.*

---

## How to Use This Rubric

**Step 1:** Run all Phase 1 automated checks. Any single failure = REVISE or ESCALATE immediately. Do not skip to Phase 4.

**Step 2:** If Phase 1 passes, run Phase 2 semantic audit.

**Step 3:** If Phase 2 passes, run Phase 3 archetype consistency check.

**Step 4:** If Phases 1-3 pass, run Phase 4 human judgment scoring.

**Step 5:** Apply the verdict formula.

---

## Phase 1: Automated Checks (Objective, Binary)

All 12 checks must pass. Each is a FAIL if the condition is not met.

### Banned String Checks (grep: must return 0 matches)

| # | Pattern to grep | Why it fails |
|---|---|---|
| 1.1 | `ease-in-out` in any .tsx, .ts, or .css file | The generic web app easing tell |
| 1.2 | `duration-300` in any .tsx, .ts, or .css file | The generic 300ms timing tell |
| 1.3 | `ease-linear` (outside linear() spring definitions) | Generic linear timing |
| 1.4 | `box-shadow.*rgba(0,0,0,0.1)` | Tailwind shadow-md applied globally |
| 1.5 | `grid-template-columns: repeat(3, 1fr)` with 3 identical direct children | The 3-box feature grid |

### Presence Checks (grep: must return 1+ matches)

| # | Pattern | File/Location | Exception |
|---|---|---|---|
| 1.6 | `SmoothScrollProvider` | `src/app/layout.tsx` | BRUTALIST, TRUST_ENTERPRISE archetypes |
| 1.7 | `KineticText` | Anywhere in `src/` | BRUTALIST archetype |
| 1.8 | `MagneticButton` | Anywhere in `src/` | BRUTALIST archetype |
| 1.9 | `@import "./craft-tokens.css"` or `@import "craft-tokens.css"` | `src/app/globals.css` (must be first import) | None |

### Structural Checks (inspect: binary)

| # | Check | FAIL condition |
|---|---|---|
| 1.10 | Inter font in heading selectors | `font-family` containing 'Inter' in h1, h2, or h3 CSS or Tailwind class |
| 1.11 | Uniform border radius | The same radius value applied to card elements AND button elements AND input elements |
| 1.12 | Form error states | Any `<form>` or form component with no error state variant (no error class, no aria-invalid handling, no error message element) |

## Phase 2: Semantic HTML Audit

Score: PASS if 3 or fewer violations. REVISE if 4+ violations.

| # | Check | Violation |
|---|---|---|
| 2.1 | Navigation uses `<nav>` | `<div class="nav">`, `<div class="navbar">`, or similar |
| 2.2 | Primary content uses `<main>` | No `<main>` element in the document |
| 2.3 | Content sections use `<section>` or `<article>` | All content sections are `<div>` |
| 2.4 | Footer uses `<footer>` | `<div class="footer">` |
| 2.5 | Interactive elements use `<button>` | `<div onclick>` or `<div role="button">` |
| 2.6 | Icon-only buttons have `aria-label` | `<button>` with only an icon child and no accessible label |
| 2.7 | Decorative images have empty `alt=""` | Decorative images with descriptive alt text (screen reader noise) |
| 2.8 | Informative images have meaningful `alt` text | Informative images with `alt=""` |

## Phase 4: Human Judgment Scoring

Score each criterion 1-5 using the anchors. Record the score.

### Criterion 1: Compositional Integrity
**Question:** Does the layout have a deliberate visual hierarchy beyond centering?

| Score | Evidence |
|---|---|
| 1 | All content is centered. Sections are stacked symmetrically. No axis, tension, or asymmetry of any kind. |
| 2 | One or two elements are not centered, but the overall layout still feels default and symmetric. |
| 3 | Some asymmetry exists but feels accidental rather than designed. The layout could have been generated by any Tailwind starter. |
| 4 | Clear compositional axis (not centered). At least one section breaks the expected layout pattern. The eye moves with intent. |
| 5 | The layout has a deliberate point of view. Multiple sections use the non-centered axis consistently. The composition would require a decision to reproduce. |

### Criterion 2: Typographic Differentiation
**Question:** Does the font pairing create genuine personality?

| Score | Evidence |
|---|---|
| 1 | All text uses Inter or Plus Jakarta Sans at 400 and 600 weights. No personality, no contrast. |
| 2 | A non-Inter font is used but only as a slight variation (e.g., DM Sans in place of Inter). No significant personality difference. |
| 3 | A second font appears but the pairing feels safe or accidental. A reader would not notice or remember the typography. |
| 4 | The font pairing creates genuine contrast and personality. A non-designer can sense the intentionality even if they cannot name the fonts. |
| 5 | The typography is the site's primary personality vector. Removing the text would remove the site's character. The pairing is specific, not generic. |

### Criterion 3: Motion Choreography
**Question:** Does motion reinforce hierarchy or just decorate?

| Score | Evidence |
|---|---|
| 1 | All elements enter with the same fade-in at the same speed. No relationship between motion and content hierarchy. Motion is decoration. |
| 2 | The hero has more animation than other sections but the difference is quantitative (more complex, same type). |
| 3 | Hero has more dramatic motion than supporting content but the difference is degree only (faster, same direction). |
| 4 | Motion is choreographed: the headline enters differently from supporting copy, which enters differently from the CTA. The motion tells you what to look at and in what order. |
| 5 | The motion is the site's narrative guide. A user navigating with eyes closed (via screen reader) and a user watching the motion should understand the same information hierarchy. |

### Criterion 4: Archetype Consistency
**Question:** Does the full build feel like it belongs to the declared archetype?

| Score | Evidence |
|---|---|
| 1 | The build does not feel like any specific archetype. It could belong to a SaaS, an agency, a restaurant, or a law firm. |
| 2 | One or two sections feel like the declared archetype but others break the design language significantly. |
| 3 | The archetype is recognizable in most sections but breaks down in 2-3 places (e.g., a CINEMATIC build with a standard 3-column feature grid). |
| 4 | The full build is consistent with the declared archetype. Every section, component, and typographic choice feels like it belongs to the same design language. |
| 5 | A design professional shown this build would identify the archetype without being told. The build is a reference-quality example of the archetype. |

### Criterion 5: Warmth
**Question:** Would a working creative director flag this as AI-generated?

| Score | Evidence |
|---|---|
| 1 | A working creative director says "AI" or "ChatGPT" within 10 seconds of viewing. The build has multiple obvious AI-tells. |
| 2 | The creative director pauses and says "this might be AI-generated." The build is technically correct but has no point of view. |
| 3 | The creative director says "this looks professional" but does not attribute it to human design. It is anonymous: neither AI-flagged nor human-attributed. |
| 4 | The creative director says "someone made deliberate choices here." The build has a point of view even if it does not entirely convince as human-made. |
| 5 | The creative director says "who built this?" with genuine curiosity. The build has a consistent, specific point of view that reads as the product of human intent. |

---

## Verdict Formula

```
PASS requires ALL of:
  - Phase 1: 0 failures (all 12 checks pass)
  - Phase 2: 3 or fewer violations
  - Phase 3: 0 failures
  - Phase 4: All 5 criteria scored 4.0+

REVISE when:
  - Phase 1: 1-2 failures, OR
  - Phase 2: 4+ violations, OR
  - Phase 3: 1-2 failures, OR
  - Phase 4: 1-2 criteria below 4.0 (but overall average >= 3.5)
  AND cycle count < 3

ESCALATE when:
  - Phase 1: 3+ failures, OR
  - The same Phase 1 check fails twice across consecutive cycles, OR
  - Phase 4 overall average < 3.5, OR
  - Cycle count has reached 3 without PASS
```
